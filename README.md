# Recruiter System — Описание функционала

Система учёта рекрутов для Discord-сервера. Состоит из трёх связанных частей: Discord-бота, веб-сайта и внутреннего API. Все три работают как единый организм — действия в Discord моментально отражаются на сайте, и наоборот.

## 1. Как устроена система

Три компонента, которые постоянно общаются между собой:

**Discord-бот** — принимает команды, читает сообщения в каналах, начисляет баллы, отправляет уведомления в личку. Это основной инструмент администраторов и рекрутов в Discord.

**Веб-сайт** — публичная витрина системы. Рейтинг, личный кабинет, история баллов, панель суперадмина. Обновляется каждые 30 секунд без перезагрузки страницы.

**SQLite-база данных** — хранит всё: рекрутов, баллы, историю начислений, аккаунты, задания, настройки. Бот и сайт работают с одной и той же базой.

Когда администратор добавляет рекрута через `/добавить_рекрута` в Discord — система одновременно создаёт запись рекрута, аккаунт на сайте, привязывает Discord ID и отправляет данные для входа в личку. Всё одной командой.

## 2. Рекруты — базовые понятия

**Рекрут** — участник системы, идентифицируемый по Discord ID. У каждого рекрута есть:

- `user_id` — Discord ID, главный идентификатор
- `username` — Discord-тег (например, `alex#1234`)
- `display_name` — отображаемое имя на сайте и в боте (можно задать отдельно)
- `points` — текущий баланс баллов
- `status` — вычисляется автоматически из баллов
- `rank` — вычисляется автоматически из баллов
- `weekly_activity` — массив булевых значений за последние недели (зарабатывал ли рекрут что-то на этой неделе)
- `history` — полный лог всех начислений и списаний

**Баллы** никогда не хранятся как просто число, изолированное от контекста. Каждое изменение баланса записывается в историю с указанием: сколько, за что, кто добавил, когда. Текущий баланс — это всегда сумма всех записей истории.

---

## 3. Ранги

Ранг определяется автоматически по количеству баллов. Пороги настраиваются через конфиг.

| Ранг | Минимум баллов | Особенности |
|---|---|---|
| Стажёр | 0 | Начальный ранг |
| Рекрут | 30 | — |
| Агент | 75 | — |
| Элита | 150 | — |
| Куратор | 2 000 | Особый статус — подробнее ниже |

**Куратор** — это не просто ранг, это особый режим участника. Кураторы:
- Не теряют баллы при еженедельном списании
- Могут выставлять баллы за задания типа `ЗАДАНИЕ` (подробнее в разделе 6)
- Отображаются отдельно в рейтинге (другой цвет строки, пометка)
- Порог 2 000 баллов либо берётся из настройки `CURATOR_ROLE_ID` (роль в Discord), либо вычисляется автоматически по баллам — если роль не задана

## 4. Статусы

Статус показывает, насколько рекрут «жив» в системе. Обновляется автоматически при каждом изменении баллов.

| Статус | Условие | Что значит |
|---|---|---|
| Активен 🟢 | Баллы > 10 | Рекрут в норме |
| Предупреждение 🟡 | Баллы 6–10 | Скоро может попасть в опасность |
| Опасность 🔴 | Баллы 1–5 | Следующее списание может удалить |
| Удалён 💀 | Баллы = 0 | Рекрут обнулён, запись остаётся в базе |

Запись в базе при статусе «Удалён» сохраняется — рекрут не стирается физически. Это позволяет видеть историю и при необходимости восстановить баланс вручную через `/баллы`.

## 5. Discord-бот — команды

### Для всех участников

**`/рейтинг`**
Выводит таблицу рейтинга прямо в Discord. Показывает топ рекрутов с баллами, рангами и статусами. Это embed-сообщение, которое обновляется при каждом вызове с актуальными данными.

**`/профиль`**
Показывает карточку своего профиля: имя, ранг, баллы, статус, история последних начислений. Можно посмотреть профиль другого рекрута: `/профиль пользователь:@ник`. Данные берутся из базы в реальном времени.

**`/задания`**
Показывает список активных заданий, назначенных этому рекруту, с кнопками для отметки выполнения. Работает только при привязанном аккаунте сайта (подробнее в разделе 11). Каждое задание — это интерактивная кнопка, нажатие на которую отмечает выполнение и фиксирует это в базе.

**`/привязать`**
Генерирует одноразовый код привязки и присылает его в личные сообщения. Код действует 5 минут. Его нужно ввести в личном кабинете на сайте, чтобы связать Discord-аккаунт с аккаунтом на сайте. Один рекрут — одна привязка.

---

### Для администраторов

Команды доступны участникам с ролью `ADMIN_ROLE_ID` или правами `Manage Roles` в Discord.

**`/добавить_рекрута пользователь:@ник`**
Главная команда добавления. За один вызов происходит следующее:
1. В таблицу рекрутов добавляется запись с Discord ID и именем
2. Автоматически создаётся аккаунт на сайте с логином и случайным паролем (8 символов)
3. Discord ID сразу привязывается к аккаунту — рекруту не нужно делать это самому
4. Рекруту отправляется личное сообщение с логином, паролем и ссылкой на сайт

Если у рекрута закрыты личные сообщения — бот покажет данные только администратору (ephemeral-сообщение, видно только вызвавшему).

Можно задать кастомное отображаемое имя: `/добавить_рекрута пользователь:@ник имя:Позывной`. Если не задать — используется Discord-ник.

**`/убрать_рекрута пользователь:@ник`**
Удаляет рекрута из системы. Запись и история баллов удаляются из базы.

**`/баллы пользователь:@ник количество:10`**
Ручное начисление баллов. Количество может быть отрицательным — тогда баллы снимаются. Можно добавить причину: `/баллы пользователь:@ник количество:-5 причина:Нарушение`. Причина записывается в историю и отображается в логах на сайте.

**`/сжечь_баллы`**
Запускает еженедельное списание вручную (в обычном режиме оно срабатывает автоматически по расписанию). Полезно для ручного триггера или тестирования.

---

### Команды настройки

Все настройки хранятся в базе данных и применяются сразу без перезапуска.

**`/настройки списание баллы:5`** — количество баллов, которое списывается еженедельно у каждого рекрута (кроме Кураторов)

**`/настройки канал_бота id:123`** — ID канала, где другой бот пишет сообщения о принятых/отклонённых заявках. Recruiter System читает этот канал и автоматически начисляет баллы рекрутёру

**`/настройки id_бота id:123`** — ID другого бота (чтобы отличать его сообщения от обычных)

**`/настройки канал_отчётности id:123`** — канал, где рекруты пишут свои отчёты. Бот слушает только этот канал для обработки отчётов

**`/настройки канал_проверки id:123`** — закрытый канал для администраторов, куда бот присылает все обработанные отчёты с кнопками действий

**`/настройки баллы_за_отчёт кейворд:ВН баллы:5`** — задать количество баллов за конкретный кейворд в отчёте. Кейвордов может быть сколько угодно

**`/настройки показать`** — вывести все текущие настройки одним embed-сообщением

## 6. Система отчётов

Рекруты отправляют отчёты в специальный канал. Бот читает каждое сообщение и решает, что с ним делать.

### Формат отчёта

```
@Dep.REC | ORG | Имя | Ник  КЕЙВОРД
[прикреплённый скриншот]
```

Бот анализирует текст сообщения и ищет в нём кейворды. Если ни кейворда, ни вложения нет — сообщение молча игнорируется. Если кейворд есть, но нет скриншота — бот отправляет ошибку в чат и удаляет её через 30 секунд.

Бот также проверяет, является ли отправитель зарегистрированным рекрутом. Если нет — сообщение тихо игнорируется.

### Обычные кейворды (ТП, ВН, 5ВИТО и любые добавленные)

1. Бот начисляет баллы немедленно
2. На исходное сообщение рекрута ставится реакция ✅
3. В канал проверки отправляется embed с деталями отчёта и одной кнопкой: **❌ Скам — снять N баллов**. Если администратор нажимает — баллы снимаются, реакция меняется на ❌, рекруту отправляется личное сообщение об отклонении

### Кейворд ЗАДАНИЕ

Особый тип отчёта, где количество баллов не фиксировано — его вводит куратор или администратор вручную.

1. Бот **не** начисляет баллы сразу — ставит реакцию ⏳
2. В канал проверки отправляется embed с двумя кнопками:
   - **💎 Выставить баллы** — открывает модальное окно (pop-up) с полем ввода числа от 1 до 500. После ввода баллы начисляются, реакция меняется на ✅, рекруту приходит личное сообщение с суммой
   - **❌ Отклонить** — задание отклоняется без начисления, реакция меняется на ❌

Кнопка «Выставить баллы» доступна только кураторам и администраторам.

### Авто-начисление от другого бота

Если на сервере работает отдельный бот рекрутинга, который пишет сообщения вида `@рекрутёр принял заявку в семью от @кандидат`, Recruiter System может читать эти сообщения и автоматически начислять баллы упомянутому рекрутёру. Для этого нужно настроить канал и ID этого бота через `/настройки`.

## 7. Еженедельное списание

Каждый понедельник в 12:00 UTC у всех активных рекрутов автоматически списывается фиксированное количество баллов (`weeklyBurn`, по умолчанию 5).

**Кто под списание не попадает:** Кураторы (≥ 2 000 баллов или с ролью куратора).

**Что происходит пошагово:**
1. Система проходит по всем рекрутам со статусом, отличным от «Удалён»
2. У каждого вычитается `weeklyBurn` баллов
3. Если баллы упали до нуля или ниже — статус меняется на «Удалён»
4. Каждое списание фиксируется в истории с пометкой `weekly_burn`
5. Фиксируется активность недели: если рекрут за прошедшую неделю заработал хоть что-то — неделя помечается как активная в его `weekly_activity`
6. Бот публикует сводное сообщение с итогами: кто в опасности, кто удалён

Активность по неделям отображается в рейтинге на сайте в виде цветных точек рядом с именем рекрута.

## 8. Задания

Задания — это отдельная подсистема поручений, не связанная напрямую с отчётами.

**Создание задания** происходит через панель администратора на сайте. При создании указываются:
- Заголовок задания
- Описание (опционально)
- Кому назначено: конкретному рекруту (по логину) или всем сразу

**Получение и выполнение** — рекруты видят свои задания в двух местах:
- На сайте в личном кабинете
- В Discord через команду `/задания` — там они представлены как интерактивные кнопки

Нажатие кнопки в Discord или клик «Выполнено» на сайте отмечают задание как выполненное и фиксируют Discord ID выполнившего. Отметку можно снять — как через Discord-кнопку, так и через сайт.

**Просмотр выполнений** — в панели администратора для каждого задания можно развернуть список тех, кто его выполнил.

**Удаление задания** — только через панель администратора на сайте.

## 9. Веб-сайт — страницы

### Рейтинг `/`

Главная публичная страница. Доступна без авторизации.

Отображает таблицу рекрутов с: позицией, именем, рангом, баллами, статусом, активностью по неделям и прогрессом до следующего ранга. Таблица обновляется каждые 30 секунд автоматически через API.

Фильтры над таблицей: «Все», «Активные», «Предупреждение», «Опасность», «Кураторы». Сортировка по любому столбцу — клик по заголовку.

После авторизации (иконка в шапке) администраторы видят дополнительные кнопки навигации и **панель администратора** — выдвигающуюся боковую панель с тремя разделами:

**Заявки** — список пользователей, ожидающих одобрения на сайте (если кто-то зарегистрировался самостоятельно). Кнопки «Одобрить» и «Отклонить».

**Рекруты** — добавление рекрута по Discord ID прямо с сайта (без Discord-бота), переименование существующих, список всех рекрутов.

**Задания** — создание новых заданий с указанием заголовка, описания и адресата. Список всех заданий с возможностью раскрыть, кто из рекрутов выполнил каждое.

---

### Личный кабинет `/cabinet.html`

Страница рекрута. Доступна только после авторизации.

Показывает: имя, ранг, баллы, статус, историю последних начислений и активных заданий. Здесь же находится раздел привязки Discord-аккаунта — поле для ввода кода, полученного через `/привязать` в Discord.

Администраторы, вошедшие на сайте, видят личный кабинет от имени своего аккаунта.

---

### История баллов `/history.html`

Полный лог всех начислений и списаний по всем рекрутам. Доступна только администраторам.

Поддерживает фильтрацию по: конкретному рекруту, типу операции (ручное начисление, отчёт, списание, скам), диапазону дат. Каждая запись содержит: кто получил, сколько, за что, кто выдал, дату и время.

---

### Суперадмин `/superadmin.html`

Управление ролями пользователей сайта. Доступна только суперадмину (учётная запись из переменных окружения).

Показывает список всех одобренных пользователей с их текущими ролями. Позволяет повысить рекрута до администратора и наоборот. Это единственное место, где можно управлять правами доступа к сайту.

## 10. Роли и права доступа

Система различает две независимые иерархии: роли в Discord и роли на сайте.

### Роли в Discord

| Уровень | Как определяется | Доступные действия |
|---|---|---|
| Все | — | `/рейтинг`, `/профиль`, `/задания`, `/привязать` |
| Куратор | Роль `CURATOR_ROLE_ID` или ≥ 2 000 баллов | Дополнительно: выставление баллов за `ЗАДАНИЕ` |
| Администратор | Роль `ADMIN_ROLE_ID` или право `Manage Roles` | Все команды бота |

### Роли на сайте

| Роль | Как получить | Доступные страницы |
|---|---|---|
| `recruiter` | Создаётся автоматически при добавлении рекрута | Рейтинг, личный кабинет |
| `admin` | Назначает суперадмин через `/superadmin.html` | + История баллов, панель администратора |
| `superadmin` | Задаётся в переменных окружения | Все страницы + управление ролями |

Роли в Discord и роли на сайте не связаны автоматически. Администратор Discord — не обязательно администратор сайта, и наоборот. Это сделано намеренно, чтобы разграничить доступ к боту и к веб-интерфейсу.

## 11. Привязка Discord к сайту

Привязка связывает Discord ID рекрута с его аккаунтом на сайте. Она нужна для того, чтобы команда `/задания` в Discord знала, какие задания показать этому конкретному пользователю.

**При добавлении через `/добавить_рекрута`** — привязка происходит автоматически. Администратор добавляет рекрута, и его Discord ID сразу прописывается в аккаунте. Рекруту ничего делать не нужно.

**При ручной регистрации на сайте** (если рекрут создал аккаунт сам) — привязку нужно сделать вручную:
1. Рекрут пишет `/привязать` в Discord
2. Бот присылает в личку одноразовый код (действует 5 минут)
3. Рекрут вводит код в личном кабинете в разделе «Привязка Discord»
4. После успешного ввода Discord ID записывается в аккаунт

Каждый аккаунт может быть привязан только к одному Discord ID. Повторный вызов `/привязать` на уже привязанном аккаунте вернёт ошибку. Отвязать Discord можно через API или через суперадмина.

## 12. История баллов

Каждое изменение баланса любого рекрута записывается в историю. Запись содержит:

- **Рекрут** — кто получил/потерял баллы
- **Количество** — положительное (начисление) или отрицательное (списание)
- **Причина** — текстовое описание (например, `ВН`, `ЗАДАНИЕ`, `Нарушение`, `weekly_burn`)
- **Тип** — категория операции: `report` (отчёт), `manual` (ручное), `burn` (списание), `scam` (скам-отклонение), `task` (задание)
- **Кто добавил** — логин администратора или ID бота
- **Дата и время**

История не редактируется и не удаляется. Это неизменяемый аудит-лог. Физическое удаление рекрута через `/убрать_рекрута` удаляет и его историю — это единственный способ очистить записи.

Просмотреть историю можно в трёх местах:
- Личный кабинет на сайте — последние записи по своему аккаунту
