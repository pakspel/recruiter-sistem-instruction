Система учёта рекрутов для Discord-сервера. Включает Discord-бота, веб-сайт с рейтингом, панель администратора, систему приёма заявок в семью и систему отпусков.

## 1. Как устроена система

Три компонента, которые постоянно общаются между собой:

**Discord-бот** — принимает команды, читает сообщения в каналах, начисляет баллы, управляет заявками и отпусками, отправляет уведомления в личку.

**Веб-сайт** — публичная витрина системы. Рейтинг, личный кабинет, история баллов, панель суперадмина. Обновляется каждые 30 секунд без перезагрузки страницы.

**SQLite-база данных** — хранит всё: рекрутов, баллы, историю начислений, аккаунты, задания, настройки. Бот и сайт работают с одной и той же базой.

Когда администратор добавляет рекрута через `/добавить_рекрута` — система одновременно создаёт запись рекрута, аккаунт на сайте, привязывает Discord ID и отправляет данные для входа в личку. Всё одной командой.

## 2. Быстрый старт

### Требования

- Node.js 18+
- Аккаунт [Railway](https://railway.app) (или любой VPS)
- Discord-приложение с ботом

### Локальный запуск

```bash
git clone <репозиторий>
cd recruiter-sistem

npm install

# Скопируй пример .env и заполни
cp .env.example .env

npm start
```

### Создание Discord-бота

1. Открой [Discord Developer Portal](https://discord.com/developers/applications)
2. **New Application** → придумай название
3. Слева → **Bot** → **Add Bot**
4. Скопируй **Token** → вставь в `BOT_TOKEN` в `.env`
5. Включи все три **Privileged Gateway Intents**:
   - `PRESENCE INTENT`
   - `SERVER MEMBERS INTENT`
   - `MESSAGE CONTENT INTENT`
6. Слева → **OAuth2 → URL Generator**:
   - Scopes: `bot`, `applications.commands`
   - Bot Permissions: `Send Messages`, `Read Message History`, `Add Reactions`, `Embed Links`, `Read Messages/View Channels`, `Manage Messages`, `Manage Channels`, `Manage Roles`
7. Открой сгенерированную ссылку → добавь бота на сервер

> **Где взять ID?** Включи режим разработчика в Discord: **Настройки → Дополнительно → Режим разработчика**. После этого ПКМ на любом элементе (канал, роль, пользователь, сервер) даёт пункт «Копировать ID».

## 3. Переменные окружения

Скопируй `.env.example` в `.env` и заполни нужные поля.

```env
# ── Discord Bot ────────────────────────────────────────────────────────────
BOT_TOKEN=              # Токен бота из Developer Portal → Bot
CLIENT_ID=              # Application ID из General Information
GUILD_ID=               # ID твоего Discord-сервера (ПКМ по серверу → Копировать ID)

# ── Безопасность ───────────────────────────────────────────────────────────
API_KEY=                # Произвольный секретный ключ для внутреннего API
                        # Сгенерируй: openssl rand -hex 32

# ── Суперадмин сайта ───────────────────────────────────────────────────────
SUPER_ADMIN_LOGIN=      # Логин суперадмина на сайте
SUPER_ADMIN_PASSWORD=   # Пароль суперадмина (минимум 6 символов)
                        # ⚠️ Не коммить .env в репозиторий!

# ── Роли Discord ───────────────────────────────────────────────────────────
ADMIN_ROLE_ID=          # ID роли администратора в Discord
CURATOR_ROLE_ID=        # ID роли куратора (опционально)
                        # Если не задан — куратором считается тот, у кого ≥2000 баллов
RECRUIT_ROLE_ID=        # ID основной роли рекрутов (видит тикет-каналы, управляет заявками)
RECRUIT_ROLE_ID_2=      # ID второй роли рекрутов (опционально, если ролей несколько)

# ── Система заявок в семью ─────────────────────────────────────────────────
APPLICATION_REVIEW_CHANNEL_ID=  # Канал куда приходят анкеты на рассмотрение
APPROVED_ROLE_ID=               # Роль, выдаваемая одобренному кандидату
TICKET_CATEGORY_ID=             # Категория для тикет-каналов (опционально)

# ── Система отпусков ───────────────────────────────────────────────────────
VACATION_REVIEW_CHANNEL_ID=     # Канал куда приходят заявки на отпуск
VACATION_ROLE_ID=               # Роль, выдаваемая при одобрении отпуска

# ── База данных ────────────────────────────────────────────────────────────
DATA_PATH=./data.db     # Путь к SQLite-файлу
                        # На Railway: /data/recruiter.db (нужен Volume)

# ── Сервер ─────────────────────────────────────────────────────────────────
PORT=3000               # Railway выставляет сам — не трогай
SITE_URL=               # Публичный URL сайта (например, https://my-app.railway.app)
                        # Используется в DM-сообщениях рекрутам
```

## 4. Деплой на Railway

1. Зарегистрируйся на [railway.app](https://railway.app)
2. **New Project → Deploy from GitHub repo** → выбери репозиторий
3. Railway автоматически определит Node.js и запустит `node index.js`

### Настройка Volume (постоянная база данных)

Без Volume база данных сбрасывается при каждом деплое.

1. В проекте Railway → **+ New** → **Volume**
2. Mount Path: `/data`
3. В переменных окружения установи: `DATA_PATH=/data/recruiter.db`

### Переменные окружения в Railway

**Variables** → добавь все нужные переменные из таблицы выше по одной.

## 5. Первичная настройка через бота

После запуска выполни эти команды в Discord **один раз**.

### Шаг 1 — Настрой каналы отчётности

```
/настройки канал_бота        id:<ID канала другого бота>
/настройки id_бота           id:<ID другого бота>
/настройки канал_отчётности  id:<ID канала где рекруты пишут отчёты>
/настройки канал_проверки    id:<ID закрытого канала для логов>
```

> Канал проверки — закрытый канал только для администраторов. Туда приходят все обработанные отчёты рекрутов для проверки.

### Шаг 2 — Настрой баллы за отчёты

```
/настройки баллы_за_отчёт  кейворд:ТП     баллы:5
/настройки баллы_за_отчёт  кейворд:ВН     баллы:5
/настройки баллы_за_отчёт  кейворд:5ВИТО  баллы:5
```

> Кейворд `ЗАДАНИЕ` встроен отдельно — баллы за него куратор выставляет вручную.

### Шаг 3 — Настрой еженедельное списание

```
/настройки списание баллы:5
```

### Шаг 4 — Разверни панели заявок и отпусков (опционально)

Перейди в нужный канал и выполни:

```
/заявка_панель    — разместить кнопку «Подать заявку» для кандидатов в семью
/отпуск_панель    — разместить кнопку «Оставить заявку на отпуск»
```

### Шаг 5 — Проверь все настройки

```
/настройки показать
```

## 7. Система отчётности

Рекруты отправляют отчёты в специальный канал. Бот автоматически обрабатывает каждое сообщение.

### Формат отчёта

```
@Dep.REC | ORG | Имя | Ник  КЕЙВОРД
[прикреплённый скриншот]
```

**Пример:**
```
@Dep.REC | ORG | Tony | Вячеслав  ВН
[скриншот прикреплён]
```

> ⚠️ **Без скриншота баллы не начисляются** — бот пришлёт ошибку и удалит её через 30 секунд. Если отправитель не зарегистрирован как рекрут — сообщение молча игнорируется.

### Кейворды и их обработка

| Кейворд | Баллы | Логика |
|---|---|---|
| `ТП` | настраивается | Начисляются **сразу автоматически** |
| `ВН` | настраивается | Начисляются **сразу автоматически** |
| `5ВИТО` | настраивается | Начисляются **сразу автоматически** |
| `ЗАДАНИЕ` | куратор вводит вручную | Ждут оценки куратора |

Кейворды можно добавлять любые через `/настройки баллы_за_отчёт`.

### Что видит рекрут

| Реакция | Значение |
|---|---|
| ✅ | Баллы начислены (или задание принято) |
| ⏳ | Задание ждёт оценки куратора |
| ❌ | Отчёт отклонён как скам |

### Что видит администратор (канал проверки)

**Обычный кейворд (ТП / ВН / 5ВИТО и др.):**

Бот присылает embed с информацией и одной кнопкой:
- `❌ Скам — снять N баллов` — если отчёт поддельный, нажми и баллы снимутся автоматически, рекруту придёт ЛС

**Кейворд ЗАДАНИЕ:**

Бот присылает embed с двумя кнопками:
- `💎 Выставить баллы` — открывает окно ввода числа (1–500). Доступно кураторам и администраторам
- `❌ Отклонить` — отклонить без начисления

### Авто-начисление от другого бота

Если на сервере есть бот рекрутинга, пишущий сообщения вида:
```
<@рекрутёр> принял заявку в семью от <@кандидат>
<@рекрутёр> отклонил заявку от <@кандидат>
```

Укажи его канал и ID через `/настройки канал_бота` и `/настройки id_бота` — баллы будут начисляться автоматически упомянутому рекрутёру.

## 8. Система заявок в семью

Модуль позволяет кандидатам подавать анкеты прямо через Discord. Не требует доступа к сайту или знания команд — достаточно нажать кнопку.

### Как настроить

1. Укажи в `.env` переменные:
   - `APPLICATION_REVIEW_CHANNEL_ID` — закрытый канал куда приходят анкеты
   - `APPROVED_ROLE_ID` — роль, выдаваемая при одобрении
   - `RECRUIT_ROLE_ID` / `RECRUIT_ROLE_ID_2` — роли рекрутов (пингуются при новой заявке)
   - `TICKET_CATEGORY_ID` — категория для тикет-каналов (опционально)

2. В нужном канале выполни команду `/заявка_панель` — бот разместит кнопку «Подать заявку».

### Процесс подачи заявки (кандидат)

1. Кандидат нажимает кнопку **Подать заявку**
2. Открывается форма-анкета с полями:
   - Имя | Дата рождения
   - Часовой пояс | Время онлайна
   - В каких семьях были ранее
   - Откуда узнали о нас
   - О себе (свободный текст)
3. После отправки кандидат получает подтверждение: «Твоя заявка успешно отправлена! Ожидай ответа»

### Процесс рассмотрения заявки (администратор)

В канале `APPLICATION_REVIEW_CHANNEL_ID` появляется embed с данными анкеты и тремя кнопками:

| Кнопка | Действие |
|---|---|
| ✅ **Одобрить** | Выдаёт роль `APPROVED_ROLE_ID`, начисляет рекруту баллы за принятие (по умолчанию +5), отправляет кандидату ЛС с поздравлением |
| ❌ **Отклонить** | Удаляет тикет-каналы (если были), отправляет кандидату ЛС об отказе. Рекруту начисляются баллы за обработку (по умолчанию +3) |
| 🎫 **Создать тикет** | Создаёт текстовый и голосовой каналы для общения с кандидатом (видят только кандидат, рекруты и администраторы). Кнопка блокируется после нажатия |

> При одобрении или отклонении тикет-каналы удаляются автоматически.

> Баллы за одобрение и отклонение начисляются тому рекруту, который нажал кнопку, — при условии что он зарегистрирован в системе.

## 9. Система отпусков

Модуль для оформления ИС / ООС АФК (временного отсутствия). Рекруты подают заявку на отпуск, администраторы рассматривают её прямо в Discord.

### Как настроить

1. Укажи в `.env` переменные:
   - `VACATION_REVIEW_CHANNEL_ID` — закрытый канал куда приходят заявки на отпуск
   - `VACATION_ROLE_ID` — роль, выдаваемая при одобрении (например, «В отпуске»)

2. В нужном канале выполни команду `/отпуск_панель` — бот разместит кнопку «Оставить заявку».

### Процесс подачи (рекрут)

1. Рекрут нажимает кнопку **Оставить заявку**
2. Открывается форма с полями:
   - Имя, фамилия
   - Причина отсутствия
   - С какого числа (например: `29.04.2026`)
   - До какого числа (например: `13.05.2026`)
3. После отправки рекрут получает подтверждение

### Процесс рассмотрения (администратор)

В канале `VACATION_REVIEW_CHANNEL_ID` появляется embed с данными заявки и двумя кнопками:

| Кнопка | Действие |
|---|---|
| ✅ **Одобрить** | Выдаёт роль `VACATION_ROLE_ID`, отправляет рекруту ЛС с подтверждением |
| ❌ **Отклонить** | Открывает модальное окно для ввода причины отказа. Причина отправляется рекруту в ЛС |

> Embed в канале обновляется: меняет цвет, фиксирует кто принял решение. Кнопки убираются.

## 10. Веб-сайт — страницы

| Страница | URL | Доступ |
|---|---|---|
| Рейтинг рекрутов | `/` или `/index.html` | Все |
| Личный кабинет | `/cabinet.html` | Авторизованные |
| История баллов | `/history.html` | Администраторы |
| Суперадмин | `/superadmin.html` | Только суперадмин |

### Рейтинг (`/`)

Главная публичная страница. Таблица всех рекрутов с баллами, рангами, статусами и активностью по неделям. Обновляется каждые 30 секунд автоматически.

**Фильтры:** Все / Активные / Предупреждение / Опасность / Кураторы

**Для администраторов** (видны только после входа):
- `⚙ АДМИН` — открывает панель управления
- `👤 ЛК РЕКРУТА` — личный кабинет
- `📜 ЛОГИ БАЛЛОВ` — история всех начислений

**Панель администратора (⚙ АДМИН):**

- **Заявки** — одобрение / отклонение заявок на регистрацию с сайта
- **Рекруты** — добавление по Discord ID, переименование, список всех рекрутов
- **Задания** — создание новых заданий, список с возможностью раскрыть кто выполнил

### Личный кабинет (`/cabinet.html`)

Страница рекрута. Показывает профиль, историю баллов, активные задания и форму привязки Discord-аккаунта.

### История баллов (`/history.html`)

Полный лог всех начислений и списаний по всем рекрутам. Доступна только администраторам. Поддерживает фильтрацию по рекруту, типу операции и диапазону дат.

### Суперадмин (`/superadmin.html`)

Управление ролями пользователей сайта: смена ролей `recruiter` ↔ `admin`. Доступна только суперадмину.

## 11. Ранги и статусы

### Ранги (определяются автоматически по баллам)

| Ранг | Значок | Минимум баллов | Особенности |
|---|---|---|---|
| Стажёр | 🔰 | 0 | Начальный ранг |
| Рекрут | 🎯 | 30 | — |
| Агент | ⚡ | 75 | — |
| Элита | 👑 | 150 | — |
| Куратор | 🛡 | 2 000 | Не теряет баллы при списании, выставляет баллы за ЗАДАНИЕ |

**Куратор** — особый статус. Кураторы не попадают под еженедельное списание и могут вручную выставлять баллы за задания типа `ЗАДАНИЕ` через кнопку в канале проверки.

### Статусы

| Статус | Цвет | Условие |
|---|---|---|
| 🟢 Активен | Зелёный | Баллы > 10 |
| 🟡 Предупреждение | Жёлтый | Баллы 6–10 |
| 🔴 Опасность | Красный | Баллы 1–5 |
| 💀 Удалён | — | Баллы = 0 |

> Запись рекрута при статусе «Удалён» сохраняется в базе — она не удаляется физически. Это позволяет видеть историю и при необходимости восстановить баланс через `/баллы`.

## 12. Еженедельное списание

Каждый понедельник в 12:00 UTC у всех рекрутов (кроме Кураторов) автоматически списывается `weeklyBurn` баллов (по умолчанию 5).

**Пошагово:**
1. Система проходит по всем рекрутам со статусом, отличным от «Удалён»
2. У каждого вычитается `weeklyBurn` баллов
3. Если баллы упали до нуля — статус меняется на «Удалён»
4. Каждое списание фиксируется в истории с меткой `weekly_burn`
5. Фиксируется активность недели: заработал ли рекрут хоть что-то за прошедшую неделю
6. Бот публикует сводное сообщение: кто в опасности, кто удалён

Активность по неделям отображается в рейтинге на сайте в виде цветных точек рядом с именем рекрута.

**Расписание:** каждый понедельник в 12:00 UTC. Запустить вручную: `/сжечь_баллы`.

**Изменить количество:** `/настройки списание баллы:10`

## 13. Задания

Задания — отдельная подсистема поручений, не связанная напрямую с отчётами.

**Создание** — через панель администратора на сайте. При создании указывается заголовок, описание (опционально) и адресат: конкретный рекрут (по логину) или все сразу.

**Получение и выполнение** — рекруты видят свои задания в двух местах:
- На сайте в личном кабинете
- В Discord через команду `/задания` — интерактивные кнопки

Нажатие кнопки в Discord или клик «Выполнено» на сайте отмечают задание как выполненное и фиксируют Discord ID выполнившего. Отметку можно снять.

**Просмотр выполнений** — в панели администратора для каждого задания можно развернуть список тех, кто его выполнил.

**Удаление** — только через панель администратора на сайте.

## 15. Привязка Discord к сайту

Привязка связывает Discord ID рекрута с его аккаунтом на сайте. Нужна для того, чтобы команда `/задания` в Discord знала, какие задания показать.

### При добавлении через `/добавить_рекрута`

Привязка происходит **автоматически** — рекруту ничего делать не нужно.

### При ручной регистрации на сайте

1. Рекрут вводит в Discord: `/привязать`
2. Бот присылает **одноразовый код** в личку (действует 5 минут)
3. Рекрут вводит код в личном кабинете → раздел **🔗 Привязка Discord**
4. После ввода Discord ID записывается в аккаунт

> Каждый аккаунт привязывается только к одному Discord ID. Повторный `/привязать` на уже привязанном аккаунте вернёт ошибку.

## 16. История баллов

Каждое изменение баланса записывается в историю. Запись содержит:

- **Рекрут** — кто получил / потерял баллы
- **Количество** — положительное (начисление) или отрицательное (списание)
- **Причина** — текстовое описание (`ВН`, `ЗАДАНИЕ`, `Нарушение`, `weekly_burn` и т.д.)
- **Тип** — `report` / `manual` / `burn` / `scam` / `task` / `accepted` / `rejected`
- **Кто добавил** — логин администратора или ID бота
- **Дата и время**

История не редактируется. Это неизменяемый аудит-лог. Физическое удаление рекрута через `/убрать_рекрута` — единственный способ очистить записи.

Просмотреть историю можно в трёх местах:
- Личный кабинет на сайте — последние записи своего аккаунта
- Команда `/профиль` в Discord — последние начисления
- Страница `/history.html` — полный лог с фильтрами (только для администраторов)

## 17. Частые вопросы

**Бот не реагирует на отчёты**
→ Проверь настройки: `/настройки показать`. Убедись, что канал отчётности задан верно.

**Рекрут не получил ЛС при добавлении**
→ У него закрыты личные сообщения. Передай данные вручную — бот покажет их тебе в ответе (видно только тебе, ephemeral).

**Задание не показывается в `/задания`**
→ Рекрут не привязал Discord к сайту. Пусть выполнит инструкцию из раздела 15.

**Как добавить новый кейворд отчёта?**
→ `/настройки баллы_за_отчёт кейворд:НОВЫЙ баллы:7`

**Как сменить пароль суперадмина?**
→ Обнови `SUPER_ADMIN_PASSWORD` в переменных окружения и перезапусти сервер.

**База данных сбрасывается после деплоя на Railway**
→ Настрой Volume (раздел 4) и укажи `DATA_PATH=/data/recruiter.db`.

**Кнопка «Создать тикет» не работает или канал не создаётся**
→ Проверь, что у бота есть права `Manage Channels` на сервере. Если используется `TICKET_CATEGORY_ID`, убедись что ID верный.

**Роль при одобрении заявки не выдаётся**
→ Проверь что у бота есть право `Manage Roles` и что его роль стоит **выше** выдаваемой роли в иерархии сервера.

**Рекрут не получил ЛС о результате заявки на отпуск**
→ У него закрыты личные сообщения — это ограничение Discord, обойти нельзя.

**Как посмотреть, кто рассматривал заявку?**
→ Embed в канале проверки обновляется: добавляется поле «Одобрил» / «Отклонил» с тегом администратора.
